# 墨园产品与体验设计规范

## 1. 产品定义

### 一句话定位

墨园是一款把真实阅读时光转化为植物收藏与个人花圃的轻养成应用。

### 核心价值

- 它不催促用户完成任务，而是保存阅读留下的成长痕迹。
- 它不是专注计时器换皮，数据来自真实阅读行为。
- 植物是奖励，花圃是长期资产，图鉴是记忆档案。
- 用户每次打开应用，第一眼应该看到自己的花圃，而不是统计报表。

### 品牌性格

- 安静，但不寡淡。
- 精致，但不奢华。
- 有文学气质，但不复古做旧。
- 有养成反馈，但不制造焦虑。
- 植物鲜明生动，界面克制退后。

### 品牌文案

- 品牌名：墨园
- 主标语：让阅读，在时间里生长
- 同步动作：浇灌
- 新植物：相遇 / 发现
- 收藏页：图鉴
- 数据页：书案
- 完成全部图鉴：归园

避免使用“打卡失败、断签惩罚、植物死亡、落后”等焦虑性表达。

## 2. 核心养成循环

1. 用户在微信读书中正常阅读。
2. 墨园自动同步阅读分钟、书目和阅读日期。
3. 阅读分钟转化为花圃的生长进度。
4. 跨过里程碑时随机发现一株未拥有植物。
5. 用户查看植物故事，并决定是否放入花园。
6. 花圃扩建、图鉴收集和成就共同形成长期目标。

### 浇灌的产品含义

- 自动同步已经发生时，浇灌是一次可见的生长反馈，不是数据生效的必要条件。
- 点击后展示水滴、草坪微光和植物轻摆，持续 600-900ms。
- 如果跨过解锁里程碑，浇灌结束后进入新植物发现仪式。
- 没有新增阅读时，按钮显示“今日花圃已浇灌”，不重复制造奖励。
- 未授权时，按钮显示“连接微信读书”，点击进入授权流程。

## 3. 花园与花圃概念

### 基础单元

一株植物对应一块四季草坪。草坪不是彼此孤立的小岛，而是可以拼合的花圃单元。

### 方阵规格

只保留完整方阵：

| 名称 | 方阵 | 容量 | 建议开放条件 |
| --- | --- | ---: | --- |
| 初庭 | 3×3 | 9 | 初次进入 |
| 雅庭 | 4×4 | 16 | 已发现 9 株 |
| 盛庭 | 5×5 | 25 | 已发现 16 株 |
| 繁庭 | 6×6 | 36 | 已发现 25 株 |
| 满园 | 7×7 | 49 | 已发现 36 株 |

方阵经过等距投影后统一呈完整菱形。应用拥有 50 株植物，但最大展示容量为 49；第 50 株仍进入图鉴，用户可以替换花园中的任意植物。这让“收藏全部”和“如何陈设”成为两个不同目标。

### 空草坪

- 花圃始终显示当前方阵的全部草坪。
- 新用户拥有 3 株时，3×3 花圃仍显示 9 块草坪，其中 6 块为空。
- 空草坪中心保留干净种植位，不显示问号、锁或教学文字。
- 点击空草坪只显示“等待下一株植物”。

### 自动排列

- 植物从花圃中心向外按环形顺序填充，不使用从左上到右下的列表顺序。
- 奇数方阵从中心格开始；偶数方阵从中央 2×2 开始。
- 同一圈层优先保持左右、前后视觉平衡。
- 高大植物优先放后排，低矮花卉优先放前排。
- 自动模式每次新增植物只占用一个空草坪，不重排所有已有植物。

### 自定义排列

- 长按植物进入编排模式。
- 拖动植物到另一块草坪时交换位置，不允许自由像素级摆放。
- 空草坪可以接收植物；拖出花圃则回到图鉴收藏。
- 编排模式显示轻微网格轮廓和完成按钮，退出后隐藏所有编辑痕迹。
- 每次交换有轻触觉反馈，动画 180-220ms。

### 渲染规则

- 草坪之间保留约单格宽度 4%-6% 的窄缝。
- 所有草坪在底层统一绘制；所有植物在上层绘制。
- 植物按 `row + column` 做前后景排序，前排植物覆盖后排植物。
- 植物定位使用土团中心锚点，不能使用整张 PNG 外框中心。
- 植物主体宽度约为草坪单元的 52%-62%，高大植物允许向上溢出。
- 花圃整体居中，顶部不侵入品牌栏，底部不进入信息面板安全区。
- 3×3 到 7×7 使用同一投影公式，只改变缩放，不改变透视角。

## 4. 首页设计

### 页面层级

1. 顶部：墨园品牌名、标语、季节、天气。
2. 中部：占据主要视觉面积的完整花圃。
3. 花圃下方：下一次相遇进度或当前选中植物名称。
4. 底部信息板：今日阅读、连续天数、园中植物。
5. 主动作：浇灌。
6. 底部导航：花园、图鉴、书案、我的。

### 信息板

- 默认收起，只展示三项核心数据。
- 展开后展示下一次相遇、花圃方阵、自动/自定义模式。
- 信息板最大宽度与浇灌按钮一致。
- 不在首页展示累计阅读、书目总数和成就，这些属于书案与我的。

### 季节与天气

- 季节决定草坪素材、环境底色和少量装饰。
- 天气只影响环境层，不覆盖植物主体。
- 夜晚使用微暖月光，不把整个页面染成深蓝色。
- 雨雪、萤光和落叶必须低密度，不能干扰花圃识别。
- 点击天气显示更新时间和来源，不把天气当成按钮式装饰。

## 5. 图鉴设计

### 列表页

- 顶部标题“图鉴”，右侧显示“已发现 12/50”。
- 使用“全部 / 已发现 / 未发现”分段筛选；不需要复杂路径和稀有度筛选。
- 默认三列，卡片高度稳定，植物图占主要面积。
- 已发现：显示彩色植物、名称、发现日期。
- 未发现：显示低对比度轮廓、名称和“等待相遇”。
- 每一张卡片都可以点击，未发现植物也有详情状态。

### 植物详情

- 第一屏以植物大图为主，不嵌套卡片。
- 展示植物名、发现状态、简短描述、文化小传。
- 已发现植物显示发现日期、累计陪伴天数和“放入花园/移出花园”。
- 未发现植物显示剪影和“继续阅读，在新的里程碑相遇”。
- 文化小传使用文学字体和左侧细金线，不使用大段仿古装饰。
- 返回使用标准箭头图标，不使用“← 返回”文本按钮。

## 6. 书案设计

### 页面目标

书案解释“我如何阅读”，不承担花园编辑功能。

### 内容顺序

1. 本月标题和同步状态。
2. 今日阅读、连续阅读、累计阅读、阅读书目。
3. 本周阅读趋势，可左右切周。
4. 最近阅读书目及真实进度。
5. 图鉴进度作为一行辅助信息，不再单独占据大卡片。

### 图表风格

- 柱形使用品牌绿，未阅读日期使用浅灰线。
- 当前日使用暖金小标记，不用第二套大面积配色。
- 数值标签只在有阅读时显示。
- 周切换保留真实安装日期边界。

## 7. 我的设计

### 顶部身份

- 显示“我的墨园”、已发现植物数、连续阅读天数。
- 头像使用莲花品牌图标或用户选择的代表植物，不使用单独的“墨”字占位。

### 成就

- 成就采用小型印章图形，但保持平面、清晰和可扫描。
- 分类为阅读、养成、里程碑。
- 已完成显示获得日期；进行中显示单一进度，不重复显示说明。

### 设置

- 微信读书连接状态。
- 立即同步与同步时间。
- 通知设置。
- 数据与隐私。
- 关于墨园、版本和开源说明。

授权 Token 输入必须说明“仅加密保存在本机”，并使用密码输入样式。

## 8. 引导与首次使用

引导页只保留三屏，并直接使用真实植物与草坪素材：

1. 阅读会长成一座花园。
2. 连接微信读书，自动保存阅读时光。
3. 遇见植物，扩建并布置自己的墨园。

完成引导后先进入拥有 3 株植物的 3×3 初庭，再以非阻断方式提示连接微信读书。通知权限只在第一次发现新植物后解释并申请。

## 9. 品牌视觉系统

### 标志与图标

- 主标志使用现有莲花植物素材，不再创造与素材不一致的水墨线稿 Logo。
- 启动图标为墨绿色底、中央莲花、四周保留自适应图标安全区。
- Android 13 单色图标使用莲花整体剪影。
- 品牌名与图标不必在每个页面重复出现；首页和启动体验是主要品牌场景。

### 配色

| Token | 色值 | 用途 |
| --- | --- | --- |
| Ink | `#202321` | 主文字 |
| Muted | `#737873` | 次要文字 |
| Paper | `#F5F7F3` | 页面背景 |
| Surface | `#FFFFFF` | 信息板、对话框 |
| Line | `#DFE4DF` | 边框、分割线 |
| Moyuan Green | `#2F6B58` | 主动作、选中态 |
| Deep Green | `#255446` | 按下态、深色图标 |
| Soft Green | `#E1EEE7` | 轻强调背景 |
| Warm Clay | `#C77757` | 阅读状态、土壤呼应 |
| Quiet Gold | `#E5B84B` | 发现、里程碑、诗文引线 |

UI 不随季节整体换色。季节色仅进入花圃和环境，避免四个季节变成四套应用主题。

### 字体

- 霞鹜文楷：品牌名、植物名、诗文、成就名。
- 京华老宋：页面标题和少量章节标题。
- 系统无衬线字体：正文、按钮、设置、数字、导航和表单。

正文不大面积使用老宋或文楷，以保证 Android 小屏阅读清晰度。字体权重只使用 400 和 500。

### 圆角、边框与阴影

- 页面信息板、对话框、植物卡片统一 8dp 圆角。
- 小标签和分段按钮使用 6-8dp，不使用过度胶囊化。
- 默认边框 1dp，颜色为 Line。
- 阴影只用于悬浮信息板、对话框和拖动中的植物。
- 不使用大面积新拟态。2.5D 质感由植物与草坪承担，UI 保持平面清晰。

### 图标

- 使用统一 2dp 线性图标。
- 导航：幼苗、图鉴书册、打开的书、个人。
- 操作：返回、设置、同步、时间、通知、信息。
- 选中态使用品牌绿，不添加彩色圆底。
- 图标按钮最小触控区域 44dp，并提供无障碍说明。

## 10. 动效与反馈

- 花圃空闲：植物上下浮动不超过 2.5px，周期约 6.2s。
- Android 首页静止刷新建议 10-12fps；页面不可见时停止。
- 浇灌：水滴、草坪亮度和植物轻摆，一次性 600-900ms。
- 新植物：草坪先亮起，再从土壤中出现，整体 900-1200ms。
- 页面切换：使用系统默认 Material 动画，180-240ms。
- 列表更新：使用 DiffUtil，不闪烁、不整体刷新。
- 尊重系统“减少动态效果”设置。

## 11. 状态设计

- 未授权：首页仍可看到初始花圃，浇灌按钮引导连接微信读书。
- 离线：显示上次数据，提示“联网后自动继续”，花圃可正常浏览。
- 同步中：按钮禁用并显示明确进度，不重复发起任务。
- 同步失败：保留数据，提供原因和重试。
- 图鉴为空：不出现空白页，展示 3 株初始植物。
- 图片加载失败：显示稳定的植物轮廓占位，不改变卡片尺寸。
- 横屏和平板：花圃与信息区左右分栏，不能简单拉伸手机布局。

## 12. 设计验收标准

- 3×3 到 7×7 的花圃轮廓都必须完整、居中且无裁切。
- 任意植物不能被其他草坪覆盖。
- 植物土团中心必须落在对应草坪种植位。
- 只有植物之间允许按景深互相遮挡。
- 360dp 宽手机上所有文字、按钮和方阵控制均不溢出。
- 首页静置 30 秒无明显发热，页面不可见时无持续绘制。
- 所有 50 株图鉴卡片均可点击，已发现和未发现状态都完整。
- 首次启动、无授权、断网、同步中、同步失败和全部解锁均有明确界面。
- UI 不与植物争夺视觉中心，第一眼仍然是花圃。
