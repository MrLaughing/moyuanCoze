# 墨园 2.0 设计文稿

**版本**: v2.0  
**平台**: 墨水屏 Android 设备（灰度显示）  
**数据源**: 微信读书 API 自动同步  
**设计风格**: 2.5D PNG 贴图 + 扁平装饰元素

---

## 一、设计哲学

### 核心原则

```
简单到极致，治愈而非焦虑。
```

### 对比：旧版 vs 新版

| 维度 | 旧版（1.0） | 新版（2.0） |
|------|-----------|-----------|
| 核心指标 | 4 条解锁路径 x 倍率计算 x 季节 x 天气 | **只有 1 条：累计阅读分钟数** |
| 植物视觉 | Canvas 复杂水墨手绘（不好看） | **2.5D PNG 贴图（50 种）** |
| 背景风格 | 水墨山水（远山/散石/苔点） | **卡通花园（天空/云/竹篱/园圃）** |
| 园圃背景 | 无 | **四季背景 PNG**（春/夏/秋/冬，空白地块展示） |
| 惩罚机制 | 枯萎/恢复循环（产生负罪感） | **无惩罚**（长期不读... 休眠，仅视觉变淡） |
| 路径系统 | 积墨/秉烛/岁寒/寻芳 + 倍率 x1.2 | **彻底移除** |
| 画面密度 | 27 株挤在一起，看不清 | 展示 **6~25 株**（6 种布局可切换），每个园圃清晰可见 |
| 园圃布局 | 无（植物散落于山脚） | **规整园圃格子 + 竹篱笆** |
| 植物系统 | 等级区分（LV1...LV5） | **无等级**，单张 2.5D PNG，解锁随机获取 |
| "我的"页面 | 静态文字信息 | **侧边树枝动画**（垂枝梅花风格，从左侧伸出横跨卡片，花朵密集） |
| 落日效果 | 半圆（drawArc 180度） | **完整圆形（drawCircle）+ 中心高光** |

## 二、数据模型

### 2.1 唯一核心指标

```
累计阅读分钟数 (cumulativeReadingMinutes)
```

- **来源**: 微信读书 API 全天阅读记录，每日同步一次
- **用途**: 唯一决定解锁新植物的条件
- **增长**: 只增不减，不因不阅读而扣减

### 2.2 辅助显示数据（纯展示，不参与数值计算）

| 数据 | 来源 | 展示位置 |
|------|------|---------|
| 今日阅读分钟数 | 微信读书当日统计 | 顶部状态栏 |
| 连续阅读天数 | 微信读书签到数据 | 底部面板 |
| 已读不同书目数 | 书架 API | 底部面板 |
| 图鉴收集进度 | 本地已解锁植物数 / 总数 | "我的"页面 |
| 实时天气 | Open-Meteo API + IP 定位 | 花园背景装饰 |

### 2.3 植物解锁阈值（统一按累计阅读分钟数，随机解锁）

累计阅读分钟达到阈值时，从 **50 种植物**中**随机解锁一株**（不重复，已解锁的不会再次出现）。

```
初始: 0 分钟      -> 系统赠予 3 株（随机）
第 4 株: 120 分钟  -> 随机解锁 1 株
第 5 株: 480 分钟  -> 随机解锁 1 株
第 6 株: 1200 分钟 -> 随机解锁 1 株
第 7 株: 2400 分钟 -> 随机解锁 1 株
...
第 N 株: 每递增一定分钟解锁一株
总目标: 50 种植物全部解锁
```

**注意**：植物仅用于**图鉴设定和视觉风格**，无等级区分。每张植物只有一个固定的 2.5D PNG 形态。

### 2.4 全收集后的奖励机制

当用户解锁全部 50 种植物后，后续每次达到新的解锁阈值时，弹出奖励窗口，可以选择**培育一株新苗**：

| 可选项 | 说明 |
|--------|------|
| 培育一株新苗 | 从已解锁的 50 种植物中任选一株，在园圃中新增一株副本（同种可同时摆放多株） |

**设计原则**：
- 每次达到阈值触发一次，不会堆积
- 全收集后的阅读仍然有正向反馈，不会让用户感觉"阅读失去意义"

#### 3.1.1 顶部状态栏
- **左侧**: 实时天气图标 + 装饰元素提示（"飞鸟划过" / "细雨绵绵" 等简短文案）
- **右侧**: 今日阅读分钟数

#### 3.1.2 天空区域（画面上部 ~30%）
- 浅灰色渐变背景（墨水屏用灰度层次）
- 卡通云朵：3~4 个连在一起的半圆弧
- 实时天气效果叠加：
  - 雨：云底落下的短斜线
  - 雪：小白点飘落
  - 雾：波浪横线带
  - 晴/多云：正常天空，可叠加飞鸟

#### 3.1.3 竹篱笆（天空与园圃的分界）
- 简单的横线 + 竖线交叉的栅栏图案
- 栅栏后面露出植物上半身

#### 3.1.4 园圃区域（画面中部 ~50%）

**布局方案**：支持 **6 种布局可选**，用户可在设置中选择展示多少株植物：

| 布局 | 行列 | 展示数 | 每格约 (dp) | 推荐场景 |
|------|------|-------|------------|---------|
| 简洁 | 3 x 2 | **6** | 100x100 | 6 寸以下小屏，或喜爱留白 |
| 舒适 | 3 x 3 | **9** | 85x85 | 默认推荐，7~8 寸主流设备 |
| 标准 | 4 x 3 | **12** | 75x75 | 屏占比更高，图鉴较丰富时 |
| 茂盛 | 4 x 4 | **16** | 65x65 | 10 寸+大屏，接近满铺 |
| 繁茂 | 5 x 4 | **20** | 55x55 | 满铺（解锁较多时） |
| 极限 | 5 x 5 | **25** | 50x50 | 全收集后 + 同名副本 |

**默认值：4 x 3（12 株）**

**布局切换方式**：
- 花园设置页面中以 **预览模式** 展示每种布局的样子
- 切换时实时调整格子大小，植物等比缩放
- 选择后会记住用户偏好，下次打开自动应用

**格子尺寸自适应逻辑**：
```
屏宽 / 列数 = 格子宽度
格子宽度 x 行数 + 小径间距 <= 可用高度
格子宽度随布局自动缩放，保证植物完整显示
```

**园圃设计细节**：
- 方块之间留有 **泥土小径**（浅灰底色 + 深色边框）
- 方块四角略圆（柔和视觉）
- 每株植物居中放置
- **四季背景 PNG**：每个园圃方块先绘制对应季节的 PNG（春/夏/秋/冬），再在中央叠加植物 PNG
  - 仅当植物未填满所有地块时，空白地块显示季节图片
  - 植物满铺后自动隐藏季节背景

**植物选择与排列逻辑**：

**自动模式（默认）**：
- 优先展示**最新解锁**的植物（按解锁时间倒序）
- 每次同步后自动刷新：新解锁的植物替换最早解锁的植物

**自定义模式（用户可切换）**：
- 当已解锁植物数 > 园圃格子数时，用户可进入"编辑花园"模式
- 编辑模式中，每个园圃格子显示为可点击的空位
- 底部弹出一个**备选植物列表**（已解锁但未在园圃中的植物），用户点击后即可放入指定格子
- 已放入的植物可点击**移除**回到备选列表
- 用户可**保存布局**作为当前花园配置
- 进入编辑模式的入口：花园页面顶部的编辑按钮

**自动与自定义的切换**：
- 默认自动模式（省心）
- 用户首次手动编辑后，自动切换到自定义模式并锁定
- 若用户切换回自动模式，自定义配置保留但不生效

**备选列表排序**：
- 按解锁时间倒序（最新解锁优先展示）
- 有搜索/筛选功能（可筛选植物类型）

#### 3.1.5 地面 + 底部面板
- 一条水平地面线
- 可折叠面板（默认收拢）：
  - **收拢状态**：今日阅读 . 连续天数 . 图鉴进度 摘要
  - **展开状态**：详细阅读进度条 + 下一株解锁倒计时

**侧边树枝装饰**（SideBranchTreeView）：
- 自定义 View，从卡片左侧边框**头像高度处**向右伸出
- 主轴短（wx0.12），侧枝 1~2 条（角度 +/-10...30度），子枝 1~2 条（+/-35度）

- 动画逐帧生长（30ms/帧）
- 使用 `post()` + `onMeasure` 修复首次加载布局时序问题

## 五、天气与装饰系统

### 5.1 数据源

- **定位**: IP 定位（已有 IpGeoApi）
- **天气**: Open-Meteo API（已有，WMO 标准天气码）
- **同步时机**: 每日同步时获取（SyncWorker 中已有）

### 5.2 天气分类与视觉表现

| 天气 | WMO 码 | 花园视觉元素 |
|------|--------|-------------|
| 晴 | 0 | 正常天空 + 飞鸟随机出现 + 落日/月依时段 |
| 少云/多云 | 1~3 | 天空加 1~2 朵卡通云 |
| 阴 | 45, 48 | 云层增多，整体调暗 |
| 小雨 | 51~55 | 稀疏细斜线 + 云底雨丝 |
| 中雨/大雨 | 61~65 | 密集雨丝 |
| 雷阵雨 | 80~82, 95~99 | 雨丝 + 偶尔亮线闪电 |
| 雪 | 71~75, 85~86 | 小白点飘落 + 地面白色覆盖 |
| 雾/霾 | 10~12, 28 | 横向波浪雾带，多层 |
| 大风 | （风速>5级） | 植物轻微倾斜，飘动元素加速 |

### 5.3 时间装饰元素（不受天气影响，按本地时间触发）

| 元素 | 触发时间 | 视觉画法 |
|------|---------|---------|
| **飞鸟** | 晴/多云天的 6:00~18:00 | V 形短线，从一侧划向另一侧 |
| **飘落花瓣** | 春季 3~5 月 + 非雨雪天 | 小椭圆飘落，随机方向 |
| **落叶** | 秋季 9~11 月 + 非雨雪天 | 小弧线缓缓飘落 |
| **落日/晚霞** | 17:00~19:00 | 完整圆形（红晕 3 层 + 圆形落日 + 高光斑），坐落在远山左峰 |
| **月亮** | 20:00~5:00 | 天边半圆形 |
| **萤火虫** | 夏季 20:00~23:00 | 2~3 个若隐若现的小光点 |

### 5.4 关键设计原则

**天气和时间装饰元素纯属视觉氛围，不与任何游戏数值挂钩。** 用户看到的既是实时天气，也是沉浸式的环境，不需要理解或操作任何"系统"。

## 六、数据流

```
微信读书 API 自动同步（每日）
     |
+----------------------------+
|  更新累计阅读分钟数         |
|  （只增不减）               |
+----------------------------+
     |
+----------------------------+
|  检查解锁条件                |
|  - 累计分钟 >= 植物阈值     |
|  - 随机解锁一株新植物        |
|  - 弹出通知"发现新物种！"    |
|  - 新植物默认不放入花园      |
+----------------------------+
     |
+----------------------------+
|  花园自动更新                |
|  - 已有植物维持原样          |
|  - 用户可以手动从图鉴替换     |
|  - 天气根据实时数据更新       |
+----------------------------+
```

**三种同步时机**：
1. 应用启动时自动同步
2. 后台定时同步（每 4 小时）
3. 用户手动刷新

## 七、关于长期不阅读的处理

**彻底废弃"枯萎"机制**，改为 **"休眠"** 状态：

| 不阅读时长 | 花园视觉变化 | 数值影响 |
|-----------|-------------|---------|
| 1~3 天 | 无变化 | 无 |
| 4~6 天 | 植物颜色轻微变淡（不透明度 80%） | 无 |
| 7~14 天 | 植物变淡（不透明度 60%） | 无 |
| 15~30 天 | 植物更淡（不透明度 40%），出现蜘蛛网图案 | 无 |
| 30 天+ | 维持在 40%，不再继续恶化 | 无 |
| 恢复阅读 | 1 天后恢复到 100% | 重新开始累计正常计入 |

**关键**：休眠只影响视觉（变淡），**不影响任何数值**。已解锁的植物永远属于用户，阅读数据也不会丢失。

## 八、完全废弃的概念

| 废弃概念 | 之前的作用 | 替代方案 |
|---------|-----------|---------|
| 四径（积墨/秉烛/岁寒/寻芳） | 植物分类+解锁路径+倍率计算 | 只有"累计阅读分钟"一个维度 |
| 路径倍率（x1.2 / x1.0） | 季节x天气x路径匹配=加成 | **彻底移除** |
| 季节天气加成 | 匹配路径获得额外倍率 | **只保留视觉氛围** |
| 休眠等级（4 阶段） | 不阅读的程度 | 仅视觉随天数渐变，无等级 |
| Canvas 复杂水墨植物 | 6 种手绘曲线植物 | **50 种 2.5D PNG 贴图** |
| Canvas 简笔卡通植物 | 中间态 20 种几何组合 | **50 种 2.5D PNG 贴图** |
| 水墨植物子目录（每次 5 级） | 每植物 5 等级旧 PNG | **已清理删除** |
